basic-knowledage：版本号设置--语义化版本控制

发表于 2017-11-12 | 分类 basic-knowledage | 阅读次数

语义化版本格式

版本格式：主版本号.次版本号.修订号-先行版

  • 主版本号：当修订了不兼容API时
  • 次版本号：当新增了向下兼容功能内容
  • 修订号：当对版本进行了问题修正
  • 先行号：先行版本以及编译信息

主版本号

  • 当主版本号等于0时（0.1.3）则认为当前版本为开发版本
  • 每次主版本号变更时则次版本号、修订号需要重置为0，如1.0.0
  • 1.0.0 版本则认为是开发版本与发布版本的界限

次版本号

  • 当版本功能有新增或者大量的修改改正则递增
  • 当API被启用、废弃时都需要递增

修订号

  • 当对当前版本进行问题修复是递增

先行号

  • 当在标准版本上开放新的验证类或者测试功能时新增
  • 附加编译信息
  • 在修订号后以“-”后追加，不允许有空格，只能是[0-9A-Za-z]
